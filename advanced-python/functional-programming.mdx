Python supports multiple programming paradigms, including **Functional Programming (FP)**. In functional programming, code is structured around passing functions as arguments, returning functions, and avoiding changing state or mutable data.

This paradigm relies heavily on two primary concepts: the **Declarative Approach** and **Higher-Order Functions**.

---

## Key Aspects of Functional Programming

### 1. The Declarative Approach
Unlike **Imperative Programming**, which focuses on *how* to solve a problem step-by-step (using loops, conditionals, and state updates), **Declarative Programming** focuses on *what* you want to accomplish. You express the logic of computation without describing its control flow.

#### Comparison: Doubling a List

* **Imperative Approach (How to do it):**
  ```python
  # We manually manage state, initialize a list, and loop
  numbers = [1, 2, 3, 4]
  doubled = []
  for num in numbers:
      doubled.append(num * 2)
  print(doubled)  # [2, 4, 6, 8]
  ```

* **Declarative Approach (What to do):**
  ```python
  # We declare the transformation without managing loops or state
  numbers = [1, 2, 3, 4]
  doubled = list(map(lambda x: x * 2, numbers))
  print(doubled)  # [2, 4, 6, 8]
  ```

---

### 2. Higher-Order Functions
A **Higher-Order Function** is a function that does at least one of the following:
* Takes one or more functions as arguments.
* Returns a function as its result.

Let's look at three examples of higher-order functions:

#### Example 1: Passing a Function
```python
def process_numbers(numbers, operation):
    result = []
    for num in numbers:
        result.append(operation(num))
    return result

def square(x):
    return x * x

def cube(x):
    return x ** 3

nums = [1, 2, 3, 4, 5]

print(process_numbers(nums, square))            # Output: [1, 4, 9, 16, 25]
print(process_numbers(nums, cube))              # Output: [1, 8, 27, 64, 125]
print(process_numbers(nums, lambda x: x + 10))  # Output: [11, 12, 13, 14, 15]
```
**Interpretation:** `process_numbers()` doesn't know what operation to perform. It simply calls whatever function is passed to it.

#### Example 2: Returning a Function (Closure)
```python
def multiplier(factor):
    def multiply(number):
        return number * factor
    return multiply

double = multiplier(2)
triple = multiplier(3)

print(double(10))  # Output: 20
print(triple(10))  # Output: 30
```
**Interpretation:** Here, `multiplier()` returns another function, making it a higher-order function.

#### Example 3: Real-World Example (Strategy Pattern)
Suppose you want to process employee salaries differently depending on different policies.
```python
def process_salary(salary, policy):
    return policy(salary)

def tax(salary):
    return salary * 0.9

def bonus(salary):
    return salary * 1.2

salary = 50000

print(process_salary(salary, tax))    # Output: 45000.0
print(process_salary(salary, bonus))  # Output: 60000.0
```
**Interpretation:** This demonstrates the **Strategy Pattern**: `process_salary()` remains generic, and different policies (`tax`, `bonus`) can be plugged in dynamically.

## Built-In Higher-Order Functions

### The `map()` Function
Applies a given function to all items in an iterable and returns an iterator.

```python
numbers = [1, 2, 3, 4]
squared = map(lambda x: x * x, numbers)
print(list(squared))  # [1, 4, 9, 16]
```

### The `filter()` Function
Filters elements from an iterable based on whether a function returns `True` or `False` for each element.

```python
numbers = [1, 2, 3, 4, 5, 6]
even_numbers = filter(lambda x: x % 2 == 0, numbers)
print(list(even_numbers))  # [2, 4, 6]
```

### The `reduce()` Function
Must be imported from the `functools` module. It applies a function cumulatively to items of an iterable, reducing the sequence to a single value.

```python
from functools import reduce

numbers = [1, 2, 3, 4]
# Calculated as: ((1 * 2) * 3) * 4 = 24
product = reduce(lambda x, y: x * y, numbers)
print(product)  # 24
```

## Combined Example: Sum of Squares of Even Numbers

Let's combine the **declarative approach** and **higher-order functions** into a single pipeline. We will take a list of numbers and compute the **sum of the squares of the even numbers**.

### Step-by-Step Breakdown:
1. **Filter:** Keep only even numbers: `x % 2 == 0`
2. **Map:** Square each remaining number: `x ** 2`
3. **Reduce:** Sum the squared results: `x + y`

### Functional Code:
```python
from functools import reduce

numbers = [1, 2, 3, 4, 5, 6]

# 1. Filter even numbers -> [2, 4, 6]
evens = filter(lambda x: x % 2 == 0, numbers)

# 2. Square the numbers -> [4, 16, 36]
squares = map(lambda x: x ** 2, evens)

# 3. Sum the squares -> 4 + 16 + 36 = 56
total_sum = reduce(lambda x, y: x + y, squares)

print(total_sum)  # 56
```

### Writing it as a Single Pipeline:
You can chain these functional calls directly into a single statement:
```python
total_sum = reduce(lambda x, y: x + y, map(lambda x: x**2, filter(lambda x: x % 2 == 0, numbers)))
print(total_sum)  # 56
```

## The Pythonic Equivalent

While the nested higher-order function pipeline works, it can sometimes be difficult to read. In Python, you can achieve the exact same declarative result using a list comprehension combined with the built-in `sum()` function:

```python
numbers = [1, 2, 3, 4, 5, 6]

# Extremely clean, declarative, and readable
total_sum = sum(x**2 for x in numbers if x % 2 == 0)
print(total_sum)  # 56
```

<Tip>
  Prefer built-in functions like `sum()` and comprehensions for mathematical operations on collections, but use `map`/`filter`/`reduce` when working with functional pipelines or preexisting callback interfaces.
</Tip>
