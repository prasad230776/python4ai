## How classes work

Working with classes follows a simple pattern:

1. **Define the class** - Create a blueprint with the `class` keyword
2. **Add an `__init__` method** - Set up initial data when objects are created
3. **Create instances** - Make actual objects from your class
4. **Access the data** - Use the attributes you defined

Let's go through each step to build your first class.

## Basic class structure

Every class starts with the `class` keyword:

```python
class Account:  # Note: class names use PascalCase
    pass  # Empty class
```

## Adding an initializer

The `__init__` method runs when you create a new object:

```python
class Account:
    def __init__(self, owner, balance):
        self.owner = owner
        self.balance = balance

# Create account objects - using positional arguments
account1 = Account("Aarav", 5000.0)
account2 = Account("Amit", 12000.0)

# Or with named arguments (clearer)
account3 = Account(owner="Dev", balance=7500.0)

print(account1.owner)     # Aarav
print(account2.balance)   # 12000.0
```

<Note>
  Yes, `__init__` looks weird with those double underscores! This is called a "dunder" method (double underscore). It's just how Python works - you'll need to type it exactly like this. Don't worry, after writing it a few times it becomes second nature. Think of it as Python's special way of saying "this is the setup method".
</Note>

## Understanding self

`self` refers to the current object. It's how an object keeps track of its own data:

<Note>
  When defining methods in a class, you always include `self` as the first parameter. But when calling the method, you don't pass it - Python does that automatically!
</Note>

```python
class Account:
    def __init__(self, owner):
        self.owner = owner  # self.owner belongs to this specific account

# Using positional argument
account1 = Account("Aarav")

# Using named argument (same result)
account2 = Account(owner="Amit")

# Each account has its own owner
print(account1.owner)  # Aarav
print(account2.owner)  # Amit
```

## Another example: Dog class

Here is another simple class showing attributes and object states:

```python
class Dog:
    def __init__(self, name, breed):
        self.name = name
        self.breed = breed

dog1 = Dog("Buddy", "Golden Retriever")
print(dog1.name)   # Buddy
print(dog1.breed)  # Golden Retriever
```

## Real-world example: OpenAI model integration

Here's a practical class for AI engineering that defines an OpenAI model wrapper with relevant attributes and methods:

```python
class OpenAIModel:
    def __init__(self, api_key, model_name, temperature=0.7):
        self.api_key = api_key
        self.model_name = model_name
        self.temperature = temperature

    # Method to invoke text generation
    def invoke(self, prompt):
        print(f"Sending prompt '{prompt}' to {self.model_name} (Temp: {self.temperature})...")
        return f"[Response from {self.model_name}] Hello! I parsed your prompt: '{prompt}'"

# Create different model instances - passing model_name explicitly
gpt4_model = OpenAIModel(api_key="sk-secret-key", model_name="gpt-4o", temperature=0.2)
gpt3_model = OpenAIModel(api_key="sk-another-key", model_name="gpt-3.5-turbo", temperature=0.7)

# Call methods and access attributes
response = gpt4_model.invoke("Explain classes in Python")
print(response)            # [Response from gpt-4o] Hello! I parsed your prompt: 'Explain classes in Python'
print(gpt3_model.model_name) # gpt-3.5-turbo
```

## Class vs instance

- **Class**: The blueprint (like a recipe)
- **Instance/Object**: What you create from the class (like a cake from the recipe)

```python
# OpenAIModel is the class
# model1 and model2 are instances
model1 = OpenAIModel(api_key="key1", model_name="gpt-4o", temperature=0.7)
model2 = OpenAIModel(api_key="key2", model_name="gpt-3.5-turbo", temperature=0.1)

# Each instance has its own data
print(model1.temperature)  # 0.7
print(model2.temperature)  # 0.1

# Changing one doesn't affect the other
model1.temperature = 0.9
print(model1.temperature)  # 0.9
print(model2.temperature)  # 0.1 (unchanged)
```

<Tip>
  Always use clear, descriptive names for your classes. In Python, class names use PascalCase (like `TextProcessor` or `DataLoader`), while variable names use snake_case (like `text_processor` or `data_loader`).
</Tip>

## Common mistakes

<AccordionGroup>
  <Accordion title="Forgetting self in __init__">
    ```python
    # Wrong - forgot self parameter
    class Dog:
        def __init__(name):  # Missing self!
            name = name
    
    # Right - include self
    class Dog:
        def __init__(self, name):
            self.name = name
    ```
  </Accordion>

  <Accordion title="Not using self for attributes">
    ```python
    # Wrong - creates local variable
    class Dog:
        def __init__(self, name):
            name = name  # Just a local variable!
    
    # Right - use self
    class Dog:
        def __init__(self, name):
            self.name = name  # Instance attribute
    ```
  </Accordion>

  <Accordion title="Forgetting parentheses when creating instance">
    ```python
    # Wrong - assigns the class itself
    my_config = APIConfig  # Not an instance!
    
    # Right - call with parentheses
    my_config = APIConfig(api_key="api-key")  # Creates instance
    ```
  </Accordion>

  <Accordion title="Modifying class instead of instance">
    ```python
    # Wrong - modifies the class
    class Counter:
        count = 0
    
    c1 = Counter()
    c1.count += 1  # This affects the class!
    
    # Right - instance attribute
    class Counter:
        def __init__(self):
            self.count = 0  # Each instance gets its own
    ```
  </Accordion>
</AccordionGroup>
