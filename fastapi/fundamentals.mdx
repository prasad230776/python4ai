### FastAPI Fundamentals

FastAPI is a modern, high-performance Python web framework for building
APIs. It is based on standard Python type hints and automatically
generates interactive API documentation.

## Learning Objectives

After completing this chapter, you will be able to:

-   Install and configure FastAPI
-   Create and run your first API
-   Understand routing and HTTP methods
-   Work with path and query parameters
-   Accept request data using Pydantic models
-   Return structured responses
-   Use status codes effectively
-   Validate request data
-   Explore the automatically generated API documentation

## Your First FastAPI Application

Create a file named `main.py`.

``` python

app = FastAPI()

@app.get("/")
def home():
    return {"message": "Welcome to FastAPI"}
```

## Routing

A **route** connects an **API endpoint** to a **Python function**.

When a client sends a request to an endpoint, FastAPI finds the matching route and executes the corresponding Python function.

General syntax:

```python
@app.<http_method>("endpoint")
def function_name():
    ...
```

For example:

```python
@app.get("/students")
def get_students():
    return {"message": "Getting all students"}
```

Here:

- `@app.get` → Handles **GET** requests.
- `/students` → API endpoint.
- `get_students()` → Python function executed for this endpoint.

When a client sends the following request,

```http
GET /students
```

FastAPI automatically executes:

```python
get_students()
```

The function processes the request and returns a response to the client.

### Request Flow

```text
HTTP Request
GET /students
      │
      ▼
Route
@app.get("/students")
      │
      ▼
Python Function
get_students()
      │
      ▼
HTTP Response
```

### Common Routes for a Student API

Retrieve all students.

```python
@app.get("/students")
def get_students():
    return {"message": "Getting all students"}
```

Create a new student.

```python
@app.post("/students")
def create_student():
    return {"message": "Student created"}
```

Update an existing student.

```python
@app.put("/students/{student_id}")
def update_student(student_id: int):
    return {"message": f"Updating student {student_id}"}
```

Delete a student.

```python
@app.delete("/students/{student_id}")
def delete_student(student_id: int):
    return {"message": f"Deleting student {student_id}"}
```

> **Remember:** A route is simply a mapping between an **API endpoint** and a **Python function**.

## Path Parameters

A **Path Parameter** is a variable that forms part of the endpoint. It is commonly used to identify a specific resource.

```python
@app.get("/students/{student_id}")
def get_student(student_id: int):
    return {"student_id": student_id}
```

Request

```http
GET /students/101
```

FastAPI automatically extracts the value from the URL and passes it to the function.

```python
student_id = 101
```

Path parameters are typically used to access a single resource.

Examples:

```text
/students/101
/products/25
/orders/5001
```

---

## Query Parameters

**Query Parameters** are values sent after the `?` in the URL. They provide additional information to the server.

```python
@app.get("/students")
def get_students(course: str, semester: int):
    return {
        "course": course,
        "semester": semester
    }
```

Request

```http
GET /students?course=CSE&semester=4
```

FastAPI automatically assigns

```python
course = "CSE"
semester = 4
```

Query parameters are commonly used for:

- Searching
- Filtering
- Sorting
- Pagination

## Request Body

When creating or updating a resource, the client sends data in the **request body**.

FastAPI uses **Pydantic models** to define the expected structure of the data.

```python
from pydantic import BaseModel

class Student(BaseModel):
    name: str
    age: int
    course: str
```

Use the model in a route.

```python
@app.post("/students")
def create_student(student: Student):
    return student
```

Request Body

```json
{
    "name": "Rahul",
    "age": 20,
    "course": "CSE"
}
```

FastAPI automatically converts the JSON data into a `Student` object.

---

## Response Models

A **Response Model** defines the structure of the data returned to the client.

```python
from pydantic import BaseModel

class StudentResponse(BaseModel):
    id: int
    name: str
    age: int
    course: str
```

```python
@app.post("/students", response_model=StudentResponse)
def create_student(student: Student):
    return {
        "id": 101,
        **student.model_dump()
    }
```

Only the fields defined in `StudentResponse` are returned to the client.

---

## Status Codes

Every HTTP response includes a **status code** indicating the result of the request.

```python
from fastapi import status

@app.post("/students", status_code=status.HTTP_201_CREATED)
def create_student(student: Student):
    return student
```

Common status codes:

| Code | Meaning |
|------|---------|
| 200 | OK |
| 201 | Created |
| 204 | No Content |
| 400 | Bad Request |
| 401 | Unauthorized |
| 403 | Forbidden |
| 404 | Not Found |
| 422 | Validation Error |
| 500 | Internal Server Error |

## Request Validation

One of FastAPI's most useful features is **automatic request validation**.

```python
from pydantic import BaseModel, Field

class Student(BaseModel):
    name: str
    age: int = Field(gt=0)
    course: str
```

Invalid Request

```json
{
    "name": "Rahul",
    "age": -5,
    "course": "CSE"
}
```

Since `age` must be greater than `0`, FastAPI automatically returns a validation error without executing the route.

This helps ensure that only valid data reaches your application.

FastAPI automatically validates incoming data and returns validation
errors when the request is invalid.
