# Grocery Management API

In this project, you will build a simple **Grocery Management REST API** using FastAPI. Along the way, you will learn the fundamental concepts required to develop REST APIs, including routing, request handling, request validation, and response generation.

By the end of this project, you will be able to create your own APIs using FastAPI.

---

## Project Setup

**Task:** Create the workspace folder, set up a Python virtual environment, and install FastAPI and Uvicorn.

### Step 1: Create the Project Folder
Create a directory named `grocery-api` and change into it.
```bash
mkdir grocery-api
cd grocery-api
```

### Step 2: Create a Virtual Environment
Create a clean environment for your dependencies.
```bash
python -m venv .venv
```

### Step 3: Activate the Virtual Environment
Activate the environment so that any package you install is isolated to this project.

**Windows**
```bash
.venv\Scripts\activate
```

**macOS / Linux**
```bash
source .venv/bin/activate
```

### Step 4: Install Packages
Install FastAPI for building the API and Uvicorn to run the server.
```bash
pip install fastapi uvicorn
pip freeze > requirements.txt
```

## Understanding an HTTP Request

**Task:** Learn how web clients communicate with servers.

An HTTP Request is composed of:
1. **Method (Verb):** What action to perform (e.g., `GET` to read, `POST` to create).
2. **URL / Path:** Where to send the request (e.g., `/groceries`).
3. **Headers:** Metadata about the request (e.g., Content-Type).
4. **Query Parameters:** Key-value pairs in the URL (e.g., `?category=fruits`).
5. **Request Body:** The data payload sent to the server (usually in JSON format).

## HTTPException

**Task:** Handle errors gracefully when a requested item ID is not found.

If a client requests an item ID that doesn't exist, we must return an appropriate error message and status code (e.g., `404 Not Found`).

<Accordion title="Show Code">

```python
from fastapi import HTTPException

# ... (Previous code)

@app.get("/groceries/{item_id}")
def get_grocery_by_id(item_id: int):
    if item_id not in groceries:
```

## Complete Application

**Task:** Consolidate all the steps into a single, fully functional `main.py` file.

Below is the complete code containing all imports, schemas, endpoints, and error handling.

<Accordion title="Show Code">

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import Optional

app = FastAPI()

# Pydantic Schemas
class GroceryItem(BaseModel):
    name: str
    price: float
    quantity: int
    category: str

class GroceryResponse(BaseModel):
    id: int
    item: GroceryItem

# Local Data Store
groceries = {
    1: {"name": "Apple", "price": 1.5, "quantity": 100, "category": "Fruits"},
    2: {"name": "Milk", "price": 2.2, "quantity": 50, "category": "Dairy"}
}

@app.get("/")
def home():
    return {"message": "Welcome to Grocery Management API"}

@app.get("/groceries")
def get_all_groceries(category: Optional[str] = None):
    if category:
        filtered = {id: item for id, item in groceries.items() if item["category"].lower() == category.lower()}
        return filtered
    return groceries

@app.get("/groceries/{item_id}")
def get_grocery_by_id(item_id: int):
    if item_id not in groceries:
        raise HTTPException(status_code=404, detail="Grocery item not found")
    return groceries[item_id]

@app.post("/groceries", response_model=GroceryResponse)
def create_grocery_item(item: GroceryItem):
    new_id = max(groceries.keys()) + 1 if groceries else 1
    groceries[new_id] = item.model_dump()
    return {"id": new_id, "item": groceries[new_id]}
```
