Because Streamlit executes your script from top-to-bottom on every user interaction, regular Python variables are reset to their default values. 

**Session State** solves this by letting you share variables across reruns for the duration of a user session.

---

## What is Session State?

Session State acts as a dictionary-like container (`st.session_state`) unique to each user browser connection.

```python
import streamlit as st

# Checking st.session_state keys
st.write("Current Session State keys:", list(st.session_state.keys()))
```

---

## Initializing and Accessing Variables

You must initialize state variables once so they don't get reset on subsequent runs:

```python
import streamlit as st

# 1. Initialize once
if "clicks" not in st.session_state:
    st.session_state.clicks = 0

# 2. Modify value
if st.button("Click to Increment"):
    st.session_state.clicks += 1

# 3. Access value
st.write(f"Button clicks: {st.session_state.clicks}")
```

---

## Resetting State

To reset state values back to defaults, you can assign key-value pairs directly or clear them using `.clear()`:

```python
if st.button("Reset Counter"):
    st.session_state.clicks = 0
```
