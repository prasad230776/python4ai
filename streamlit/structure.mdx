Streamlit operates differently than typical Python programs. To write effective apps, you must understand its execution flow and rerun behavior.

---

## The Script Rerun Mechanism

The most fundamental rule of Streamlit development is:

> [!IMPORTANT]
> **Rerun on Interaction:** Whenever a user interacts with *any* widget (types a character, clicks a button, toggles a checkbox), Streamlit reruns your entire Python script from **top to bottom**.

### Why does it do this?
Because Streamlit does not keep a continuous active state of standard local variables in memory. By running from the top, it ensures that your layout reflects the newest inputs instantly.

---

## Basic Application Structure

A standard Streamlit application script runs line-by-line:

```python
# app.py
import streamlit as st

# 1. Page title renders
st.title("My App")

# 2. Input widget reads value
name = st.text_input("Enter name:")

# 3. Code processes value
if name:
    st.write(f"Hello, {name}!")
```

---

## The Entry Point

When you run:
```bash
streamlit run app.py
```
Streamlit initializes a local web server and mounts the file `app.py` as the entry point. Any modifications to `app.py` while the server is active will trigger a prompt in the browser asking to **Source Reload** or **Always Rerun** upon saving.
